# Workbench-only 实现约束

本文件记录当前版本的工程边界：用户入口只保留 Qt/QML Workbench，所有历史文本入口要求已移除。

## 1. 禁止事项

- 不再新增文本入口层。
- 不在 QML 中实现扫描、规则、评分、证据匹配或报告拼接。
- 不把 LLM 输出写入最终评分。
- 不覆盖原始项目文件。
- 不生成虚假用户、营收、合作、专利、实验结果或市场数据。

## 2. 入口边界

用户流程从 `apps/contest-workbench/main.cpp` 启动。该入口只负责注册 `CompileController` 并加载 QML。

`CompileController` 可以调用：

- `AuditPipeline`
- `DiffVerifier`
- `MarkdownReporter`
- `JsonReporter`
- `LlmBrain`

`CompileController` 不实现业务规则，只做参数收集、状态转换和 QML 模型桥接。

## 3. 模块边界

```text
contest_core
  数据模型、loader、inventory、text、cpir、claim、evidence、consistency、rules、audit、repair、report

contest_agent
  ToolRegistry、PermissionGate、LifecycleHookManager、ProjectMemory、AuditSessionStore、Analyzer、AuditPipeline

contest_llm
  EndpointParser、HttpResponseParser、HttpsJsonClient、AuditPromptBuilder、LlmBrain

contest-workbench
  Qt/QML UI、CompileController、结果模型、会话模型
```

## 4. 验收重点

- fresh configure/build 不生成旧文本入口产物；
- Workbench 构建成功；
- 单元测试通过；
- acceptance 和 quality 通过；
- package 中只包含 Workbench、规则包、示例、docs、tools、README 和 LICENSE；
- 全项目文档不得重新要求旧入口层。
